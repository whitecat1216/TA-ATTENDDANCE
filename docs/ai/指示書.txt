【目的】
このフォルダは、就業管理システム TA プロジェクトに関する作業指示・履歴管理のために使用します。

【基本方針】
1. 仕様変更や機能追加の依頼は、まず本フォルダの「チャット履歴.txt」に記録する。
2. 実装前に、依頼内容を「要件」「受け入れ条件」「対象画面/対象機能」に分解する。
3. 実装後は、変更点と確認結果を必ず履歴として残す。
4. 不明点がある場合は、推測実装を避けて確認事項として記載する。

【記録ルール】
- 1タスクにつき1ブロックで履歴を残す。
- 日付・依頼内容・実施内容・結果・未対応事項を最低限記入する。
- 画面名や機能名は統一した名称で記載する（例：打刻・勤怠入力、シフト管理）。

【優先実装機能（現時点）】
- 打刻（出退勤）
- シフト管理
- 休暇管理
- 勤怠集計照会
- 従業員/権限管理
- CSVエクスポート（給与連携）

【保存先】
- チャット履歴: docs/history/チャット履歴.txt
- 環境構築手順: docs/setup/環境構築手順書.txt

【補足】
- 本プロジェクトは React + Electron + Supabase(PostgreSQL) 構成。
- ドキュメントは docs 配下で用途別（ai/setup/history）に管理する。
