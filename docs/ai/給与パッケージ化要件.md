# 人事給与パッケージ化要件（TA）

## 1. 目的
- 就業管理システム TA を、複数企業に展開可能な人事給与パッケージとして成立させる。
- 会社固有差分は設定で吸収し、コード改修を最小化する。

## 2. 対象範囲（MVP）
- 従業員マスタ管理
- 部門・役職・雇用区分管理
- 入社・退職・異動の人事イベント管理
- 勤怠データの締め処理
- 法定/所定の残業・深夜・休日の集計
- 給与システム向けCSV出力
- 申請/承認ステータスを反映した確定データ作成

## 3. 非対象（このフェーズでは実施しない）
- 社会保険/税計算そのもの
- 人事評価機能
- 採用管理
- モバイル専用UIの全面最適化

## 4. 機能要件
1. 人事管理
- 従業員の基本情報、所属部門、役職、雇用区分を管理可能
- 入社日、退職日、異動履歴を保持可能
- 有効/無効、在籍状態を管理可能

2. 組織管理
- 部門の階層管理が可能
- 管理者、マネージャー、一般従業員などの権限区分を扱える

3. 締め設定
- 月次締め日、起算日、締め時刻を設定可能
- 再締め時の差分検出（再計算対象の明示）

4. 集計エンジン
- 日次明細から月次集計を生成
- 丸め設定（15分/30分/60分、切上げ/切捨て/四捨五入）
- 深夜帯（22:00-05:00）区分

5. CSV出力
- 出力項目マッピングを設定で変更可能
- エンコーディング（UTF-8/Shift_JIS）切替
- ヘッダー有無、区切り文字を設定可能

6. 承認連携
- 未承認レコードは締め対象外にできる
- 承認済みのみで確定ファイルを作成可能

7. 監査ログ
- 従業員情報変更の履歴を記録
- 締め実行者、実行時刻、対象期間、件数、再実行有無を記録

## 5. データ要件
- 新規テーブル候補:
  - employee_positions（役職マスタ）
  - employee_status_history（入退社・異動履歴）
  - organization_settings（組織設定）
  - payroll_periods（締め期間）
  - payroll_results（月次集計結果）
  - payroll_export_jobs（CSV出力履歴）
  - payroll_settings（会社別設定）

## 6. 受け入れ条件
- 従業員の所属・役職・雇用区分変更を履歴付きで管理できる
- 入社・退職・異動が月次集計対象に正しく反映される
- 指定締め日の月次集計がUIとCSVで一致する
- 会社A/Bで異なるCSVフォーマットを設定だけで切替できる
- 未承認データ除外ON時に、対象外件数が確認できる
- 同一期間再実行時に差分件数を表示できる

## 7. 実装順序（推奨）
1. 人事データモデル追加（従業員属性、役職、異動履歴）
2. 給与データモデル追加（締め期間、集計結果、出力履歴、設定）
3. 集計ロジックをサービス層へ分離
4. 締め処理API（またはユースケース）追加
5. CSVエクスポート設定化
6. 監査ログ記録
7. 画面統合（人事管理、締め実行、結果確認）

## 8. リスクと対策
- リスク: 会社ごとの差分が増えて分岐地獄になる
- 対策: 設定駆動 + 変換アダプタ層を採用

- リスク: 認証/権限未整備で本番運用できない
- 対策: 本番前に Supabase Auth とロール認可を必須タスク化

- リスク: 人事イベントの履歴不整合で集計結果がずれる
- 対策: 異動・退職を履歴テーブルで管理し、集計対象期間に応じて参照する
