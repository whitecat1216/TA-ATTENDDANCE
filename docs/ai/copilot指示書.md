# Copilot 作業指示書（TAプロジェクト）

## 目的
- 就業管理システム TA の実装・修正作業を、要件ベースで安全かつ一貫して進める。

## 前提
- 技術構成: React + TypeScript + Vite + Electron + Supabase(PostgreSQL)
- 履歴管理: docs フォルダ内のファイルを参照・更新する。
- 実装計画: docs/ai/plan.md を単一の計画書として扱う。

## 作業ルール
1. 実装前に要件を3点に分解する。
- 要件
- 受け入れ条件
- 影響範囲（画面/機能/ファイル）

2. 実装前に docs/ai/plan.md を更新する。
- 要件、受け入れ条件、影響範囲、実装タスクを必ず記入する。
- 追加作業が必要な場合は、先に plan.md を更新してから実装する。

3. 既存の実装方針を優先する。
- 命名規則、型定義、既存コンポーネント構造を崩さない。
- 不要な大規模リファクタは行わない。

4. 変更は最小単位で行う。
- 直接要求された範囲を優先。
- 無関係な整形・変更は避ける。

5. 実装後に検証する。
- 型エラー、ビルドエラー、画面上の主要動作を確認。
- 失敗時は原因と暫定対応を明記する。

6. 記録を残す。
- docs/history/チャット履歴.txt に、日付・依頼内容・実施内容・結果・未対応事項を追記する。
- docs/ai/plan.md の進捗チェックと検証項目を更新する。

## 出力フォーマット
- 変更概要
- 変更ファイル一覧
- 動作確認結果
- 残課題（あれば）

## 優先対応機能
- 打刻（出退勤）
- シフト管理
- 休暇管理
- 勤怠集計照会
- 従業員/権限管理
- CSVエクスポート（給与連携）

## 認証に関する暫定方針
- 当面は認証実装を後回しにしてもよい。
- 認証が未実装でも進められる範囲（画面・CRUD・集計）を優先する。
- 本番反映前に Supabase Auth 連携を必ず再導入する。
